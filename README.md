# LudoTrace for Fallout 4

A session black box for Fallout 4. Records your character state, kills, quests,
locations, inventory changes, and combat events to a structured JSONL log — one
impartial record of exactly what happened. What you do with it is up to you.

Includes an example prompt for [Claude.ai](https://claude.ai) — paste your log
and get coaching, comedy reacts, stat breakdowns, or whatever you want back.
No API key required.

## Requirements

- [F4SE](https://f4se.silverlock.org/)
- [Hydra](https://www.nexusmods.com/fallout4/mods/104159)
- Address Library for F4SE Plugins

## Install

Drop the contents of `dist/Data/` into your Fallout 4 `Data/` folder, or install
via Vortex / Mod Organizer 2.

## Usage

**1. Play.** Load any save — the mod starts recording automatically. On each save,
it writes a session log to:

```
Fallout 4\LudoTrace_Events.jsonl
```

**2. Get coaching.** Open [Claude.ai](https://claude.ai), start a new chat, and paste
this prompt followed by the contents of `LudoTrace_Events.jsonl`:

> You are a Fallout 4 playthrough coach. I'm going to paste a session log
> generated by the FO4 LudoTrace mod.
>
> My playstyle: sandbox, first playthrough, not rushing the main quest, no strong
> faction preference yet. I want to enjoy the world more deeply, not complete it faster.
>
> Based on the log, give me:
> 1. The 3 most impactful things I could do next session
> 2. Any "you have X but aren't using it" missed connections
> 3. One settlement tip if relevant
> 4. One collectible worth grabbing that's near my current location or active quests
>
> Be specific — reference actual item names, perk names, and locations from the log.
> Don't suggest main quest progression or push faction choices.

The prompt is also included as `coaching_prompt.md` in the release zip.

**Example output:**

> **You have 14 Mini Nukes and fought a Legendary Alpha Deathclaw for 37 in-game
> minutes without using one.** A Fat Man shot would have ended that fight in seconds.
>
> **Your Perception was knocked to 1 three times** during the Vault 88 mole rat
> fights — that's the DLC disease stacking, not limb damage. Visit a doctor before
> your next session or your VATS accuracy is gutted.
>
> **912 Plasma Cartridges, zero used this session.** With INT 8 you almost certainly
> have Science! ranks. A plasma weapon shreds the armored targets you're struggling with.

No API key, no account needed beyond a free Claude.ai login.

## Building from source

1. Copy `tools/paths.example.bat` to `tools/paths.local.bat`
2. Fill in your `GAME` and `CK` paths
3. From WSL: `make build` (or `make build run` to also launch the game)
   From Windows: double-click `tools/compile.bat`

## Snapshot format

The log is a JSONL file — one JSON object per line. Every session begins with a
`session_start` record, ends with a `session_end` record, and has event records
in between.

**session_start / session_end** — character state snapshot (identical schema):
```json
{"type":"session_start","game_date":"2288-02-27","game_time":"01:09","level":40,"name":"Rhea",
 "special":{"S":11,"P":7,"E":4,"C":7,"I":8,"A":6,"L":7},
 "bobbleheads":["Charisma","Intelligence"],
 "ammo":[
   {"name":".308 Round","form_id":"0x0001F66B","count":1001},
   {"name":"Shotgun Shell","form_id":"0x0001F673","count":238},
   {"name":"Fusion Cell","form_id":"0x000C1897","count":74},
   {"name":"Mini Nuke","form_id":"0x000E6B2E","count":14}
 ],
 "aid":[
   {"name":"Stimpak","form_id":"0x00023736","count":248},
   {"name":"RadAway","form_id":"0x00023742","count":7},
   {"name":"Jet","form_id":"0x000366C5","count":3},
   {"name":"Stealth Boy","form_id":"0x0004F4A6","count":2}
 ]}
```
Ammo and aid only include non-zero counts. Diffing start vs end gives session deltas.

**Events** — appended as they happen during the session:
```jsonl
{"type":"location","name":"The Slog","game_time":"01:12"}
{"type":"near_collectible","category":"bobblehead","name":"Strength","location":"Mass Fusion Building","game_time":"05:39"}
{"type":"kill","target":"Feral Ghoul Reaver","killer":"","game_time":"05:30"}
{"type":"stat","stat":"Creatures Killed","value":826,"game_time":"05:34"}
{"type":"quest_stage","quest":"The Road to Freedom","stage":20,"game_time":"03:14"}
{"type":"quest","name":"Reunions","state":"completed","game_time":"04:45"}
{"type":"level","to":5,"perks":[{"name":"Rifleman","rank":1},{"name":"Sneak","rank":2}],"game_time":"05:35"}
{"type":"menu_mode","menu":"PipboyMenu","state":"Entered","game_time":"01:16"}
{"type":"used","category":"healing","item":"Stimpak","game_time":"05:41"}
{"type":"used","category":"chem","item":"Jet","game_time":"05:52"}
{"type":"av_change","av":"Perception","from":7,"to":8,"game_time":"05:35"}
{"type":"combat","actor":"Raider","state":2,"game_time":"05:30"}
{"type":"limb","actor":"Brotherhood Knight","state":"Crippled","game_time":"05:30"}
```

`killer` is empty when the player lands the kill directly; a companion name
indicates companion-assisted kills. `av_change` fires only for SPECIAL stats
(e.g. bobblehead pickup). `combat.state` 1 = in combat, 2 = out of combat.
`level` includes the full perk list at the moment of level-up — diff consecutive
`level` events to see what was selected.

## Notes

- `stubs/Hydra/Events.psc` is a minimal compilation stub for `Hydra:Events`.
  Papyrus Compiler v2.8.0.4 does not support struct arrays; this stub strips
  the array fields from `*Args` structs so the file compiles cleanly.
  Hydra's real compiled `.pex` is used at runtime — the stub is compile-time only.
